# Status Plugin - 在线时间统计插件

这个插件用于统计机器人的每日在线时间，并定时发布统计结果。

## 功能特性

- **实时统计**: 自动统计机器人的在线时间和重连次数
- **自动发布**: 每天定时发布昨日的在线时间统计报告
- **手动触发**: 支持通过特定标签手动获取统计信息
- **数据持久化**: 所有统计数据保存在数据库中，重启不丢失
- **详细统计**: 包含总在线时间、在线率、重连次数、最长连续在线时间等指标

## 配置说明

### 基本配置

```yaml
enabled: true                    # 是否启用插件
priority: 80                     # 插件优先级（数字越大优先级越高）
```

### 自动发布配置

```yaml
auto_post_enabled: true         # 是否启用自动发布每日统计
post_hour: 0                    # 自动发布时间 - 小时 (0-23) 
post_minute: 0                  # 自动发布时间 - 分钟 (0-59)
```

### 手动触发配置

```yaml
manual_trigger_enabled: true    # 是否启用手动触发功能
status_tag: "#status"           # 手动触发标签
```

### 标签配置

```yaml
auto_tag: "#dailystatus"        # 自动发布时使用的标签
```

### 消息模板配置

可以自定义统计报告的格式，支持以下变量：

- `{online_hours}`: 在线小时数
- `{online_minutes}`: 在线分钟数  
- `{online_percentage}`: 在线率百分比
- `{reconnection_count}`: 重连次数
- `{max_continuous_hours}`: 最长连续在线小时数
- `{max_continuous_minutes}`: 最长连续在线分钟数
- `{date}`: 统计日期
- `{auto_tag}`: 自动发布标签

## 使用方法

### 自动发布

插件会在每天配置的时间点（默认0点）自动发布昨日的在线时间统计报告。

### 手动触发

在任何时候发送包含配置的触发标签（默认`#status`）的消息，插件会立即回复昨日的统计报告。

例如：
```
今天机器人运行得怎么样？ #status
```

## 统计数据

插件会记录以下数据：

1. **每日在线时间**: 机器人在每日的总在线时长
2. **在线率**: 在线时间占全天时间的百分比
3. **重连次数**: 每日的网络重连次数  
4. **最长连续在线时间**: 单次连续在线的最长时间
5. **会话记录**: 每次连接会话的详细记录

## 数据库表结构

### status_daily_stats（日统计表）

- `date`: 日期
- `online_seconds`: 在线总秒数
- `reconnection_count`: 重连次数
- `max_continuous_seconds`: 最长连续在线秒数
- `last_session_start`: 最后一次会话开始时间

### status_sessions（会话记录表）

- `date`: 日期
- `start_time`: 会话开始时间
- `end_time`: 会话结束时间
- `duration_seconds`: 会话持续秒数

## 注意事项

1. 插件需要数据库支持，确保 `persistence_manager` 正常工作
2. 统计数据会在机器人重启时自动恢复
3. 自动发布有5分钟的时间窗口，避免因时钟偏差错过发布
4. 手动触发功能可以随时查看昨日统计，不影响自动发布计划
