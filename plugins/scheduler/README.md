# Scheduler Plugin (定时调度插件)

这是一个用于定时发送预设内容的插件，支持多种发送场景和时机。

## 功能特性

### 1. 每日定时发送
- 可配置每天固定时间发送消息
- 支持自定义发送时间（小时和分钟）
- 支持配置多条消息，随机选择发送
- 自动记录发送状态，避免重复发送

### 2. 节日自动发送
- 支持按日期配置节日消息
- 支持多种日期格式：
  - 阳历格式：`01-01`、`12-25`
  - 中文格式：`1月1日`、`12月25日`
  - 自定义节日名称：`春节`、`中秋节`
- 每个节日可配置单条消息或多条随机消息

### 3. 启动时发送
- 机器人启动完成后自动发送欢迎消息
- 支持配置多条启动消息，随机选择
- 可以让用户知道机器人已重新上线

### 4. 断线重连发送
- 检测网络重连后发送提示消息
- 通过时间间隔判断是否为重连场景
- 提醒用户可能存在的消息延迟

## 配置说明

### 基本配置
```yaml
enabled: true          # 启用插件
priority: 90           # 插件优先级
daily_enabled: true    # 启用每日发送
holiday_enabled: true  # 启用节日发送
startup_enabled: true  # 启用启动发送
reconnect_enabled: true # 启用重连发送
```

### 每日发送配置
```yaml
daily_hour: 9      # 发送时间 - 小时 (0-23)
daily_minute: 0    # 发送时间 - 分钟 (0-59)
daily_messages:    # 消息列表（随机选择）
  - "早上好！新的一天开始了~"
  - "今天也要加油哦！"
```

### 节日发送配置
```yaml
holiday_messages:
  "01-01":         # 元旦
    - "新年快乐！🎉"
  "12-25":         # 圣诞节
    - "圣诞快乐！🎄"
  "春节":          # 自定义节日
    - "春节快乐！🧧"
```

### 启动和重连消息
```yaml
startup_messages:
  - "系统启动完成！"
  - "我回来了！"

reconnect_messages:
  - "网络连接恢复！"
  - "重连成功！"
```

## 使用方法

1. 确保插件已启用并正确配置
2. 插件会根据配置自动在相应时机发送消息
3. 可通过修改配置文件调整发送内容和时间
4. 插件会自动处理重复发送的防护

## 工作原理

### 每日发送
- 使用 `on_auto_post` 钩子检测当前时间
- 比较当前时间与配置的发送时间
- 通过数据库记录防止同一天重复发送

### 节日发送
- 在 `on_auto_post` 中检查当前日期
- 匹配配置的节日日期格式
- 优先发送节日消息（优先级高于每日消息）

### 启动发送
- 使用 `on_startup` 钩子
- 记录启动时间到数据库
- 调度启动消息发送

### 重连检测
- 通过 `on_message` 钩子监听消息
- 比较当前时间与上次启动时间
- 判断时间间隔是否符合重连特征

## 数据持久化

插件使用以下数据库键来保存状态：
- `last_daily_send_date`: 最后一次每日发送的日期
- `last_startup_time`: 最后一次启动时间
- `pending_messages`: 待发送的消息队列

## 注意事项

1. 确保机器人有发帖权限
2. 节日消息优先级高于每日消息
3. 重连检测基于时间间隔，可能存在误判
4. 建议定期检查数据库中的持久化数据
5. 消息内容支持 emoji 和 Misskey 格式语法

## 故障排除

### 消息未发送
1. 检查插件是否启用
2. 确认时间配置是否正确
3. 检查数据库连接是否正常
4. 查看日志中的错误信息

### 重复发送
1. 检查数据库中的发送记录
2. 确认时间判断逻辑
3. 重启机器人重置状态

### 节日消息不生效
1. 检查日期格式是否正确
2. 确认当前时区设置
3. 验证配置文件语法
